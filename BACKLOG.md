# PROMPT

Goal: タスク入力にフォーカスしたら、既存のタスク一覧画面上で入力モードビューに遷移し、全画面に近い入力UIへアニメーション付きで切り替える。各プラットフォームの既存ナビゲーションと入力管理方式に沿って実装する。

Success criteria:
- タスクを連続して作成できること
- タスク作成時に日付設定およびピン留めを行えること
- タスク補完候補は常に入力欄下にリスト表示されていること
- タスク補完候補を選択したとき、タスクが作成されること
- タスク作成後も入力モードビューが維持され、入力フォーカスが外れないこと
- タスク作成後に仮想キーボードのちらつきや閉じ直しが発生しないこと
- 入力欄が blur したとき、入力モードビューが終了すること
- 戻る操作で入力モードビューを終了できること
- 入力ビュー内にピン留めボタンが追加されていること
- 入力ビュー内に日付設定ボタンが追加されていること
- ポップオーバーでのタスク補完候補が削除されていること
- タスク作成成功時はタスク作成した旨を簡易に通知(プラットフォームごとの推奨UIを利用する)されること
- apps/web、apps/android、apps/ios のすべてで同等の機能を提供できること


## ONGOING

apps/web apps/android apps/ios 全て。
タスク入力にフォーカス時、入力モードビュー(全画面に入力欄が広がる感じ。アニメーションあり)に切り替え、ピン留めや日付設定をサポート。
キーボード操作や連続入力できるのは維持。
補完候補も表示し、今までのポップオーバー補完候補表示は削除。
